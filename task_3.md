## Test Plan

1. Объект тестирования

Объектом тестирования является веб-страница "Корпоративное обучение" сайта School 21.

### Description:
Данный пункт тест плана показывает какое именно ПО мы тестируем, т.е. объект - веб-страница, десктопное приложение или мобильное приложение. И для чего это ПО вообще нужно - это позволяет определить, какой именно компонент системы подлежит тестированию.



2. Цель тестирования

Целью тестирования является проверка корректности отображения контента, работоспособности элементов интерфейса, удобства использования страницы и соответствия заявленной информации фактическому содержанию.

### Description:
Данный пункт определяет основные цели тестирования, такие как проверка корректности работы, соответствие требованиям и выявление дефектов.

3. Окружение тестирования

Тестирование проводится на персональном компьютере:
ОС: Windows 11 Pro v25H2
Разрешение: 1920х1080 (так же использование других разрешений с помощью DevTools)
Браузер: Google Chrome v146.0.7680.165
Интернет подключение: Wi-Fi, скорость Download: 40-50 Mbps,скорость Upload: 85-95 Mbps.

### Description:
Здесь определяется при каких тех. условиях проводится тестирование: ОС, устройство, браузер и другие параметры среды в зависимости от объекта тестирования.

4. Область тестирования

В область тестирования входят:
1. Текстовый контент страницы 
2. Изображения и графические элементы
3. Кнопки и ссылки 
4. Форма обратной связи 
5. Навигация внутри страницы

### Description:
В этом пункте описываются определенные элементы объекта тестирования, в рамках которых будет проводиться тестирование.

5. Подход к тестированию

Тестирование проводится вручную, с использованием чек-листов. Применимые виды тестирования:
1. Функциональное тестирование
2. UI тестирование
3. Usability тестирование

### Description:
Тут указываем как мы это будем тестировать, например чек-листами или тест-кейсами, UI проверки, ручное или автоматизированное. И соотвественно как данные проверки разделяются по тестированию, где и какие методы и типы тестирования используются.

6. Стратегия тестирования

Основной фокус тестирования направлен на:
1. Корректность отображения контента
2. Удобство восприятия информации пользователем
3. Работоспособность ключевых элементов

### Description:
Здесь определяются приоритеты и ключевые области, на которых будет сосредоточено основное влияние. 

7. Критерии начала тестирования

Начало тестирование происходит при услвовии того, что сайт доступен, страница успешно загружается и отсутствуют критические ошибки зарузки.

### Description:
Тут описывается при каких условиях мы начинаем приступать к работе, а именно к тестированию.

8. Критерии завершения тестирования

Тестирование завершается при условии того, что все пункты чек-листа закрыты, критические дефекты отсутствуют, основные элементы страницы работают корректно.

### Description:
Тут уже описывается при каких условиях можно считать что тестирование завершено.

9. Риски

К возможным рискам можно отнести: некорректное отображение страницы на различных устройствах, частичная загрузка контента (медиа), некорректная работа ссылок и кнопок.

### Description:
Этот пункт определяет потенциальные риски, которые могут повлиять на процесс тестирования и/или его результаты.
